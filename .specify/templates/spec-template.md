# Feature Spec Template

## Goal
Describe the user-visible behavior and the API boundary for the feature.

## User Stories
- As a user, I can browse the pet catalog.
- As a user, I can filter pets by species and availability.
- As an admin, I can create, update, and delete pet listings.

## Acceptance Criteria
- The feature works through the documented REST API.
- The frontend reflects the API state without direct database access.
- The UI remains responsive on desktop and mobile widths.
